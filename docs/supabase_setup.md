# DeepFlow Supabase Sync Setup

DeepFlow MVP keeps local data first. Supabase sync is optional and sends append-only learning events to the cloud when configured.

## 1. Create Supabase Project

1. Open Supabase Dashboard.
2. Create a new project.
3. Open `SQL Editor`.
4. Run `supabase/deepflow_schema.sql`.

The schema creates one table:

- `deepflow_sync_events`

The web app can insert events with the public anon key. Public reads are not enabled.

## 2. Get Project Config

In Supabase Dashboard:

1. Open `Project Settings`.
2. Open `API`.
3. Copy:
   - Project URL
   - anon public key

Do not paste the service role key into DeepFlow.

## 3. Configure DeepFlow

Open DeepFlow, then open `云端同步`.

Paste:

- Supabase Project URL
- Supabase anon public key

Click `保存并同步`.

If the network is unavailable, DeepFlow keeps events locally and retries later.

## 4. Browse Test Data

Use Supabase Dashboard as the manager interface:

1. Open `Table Editor`.
2. Select `deepflow_sync_events`.
3. Browse by `event_type`, `client_id`, `profile_id`, or `created_at`.
4. Open `event_payload` to inspect questionnaire results, identity, sessions, prompts, feedback, and level evidence.

## 5. Current Security Model

This MVP allows anonymous clients to insert sync events. It does not expose public read access.

Recommended next step for larger testing:

- Add Supabase Auth anonymous sign-in.
- Tie rows to `auth.uid()`.
- Add user-owned select policies.
- Build a private admin dashboard using authenticated admin roles.

## 6. What DeepFlow Does Not Do

The Supabase sync does not add:

- GPT API calls
- PDF parsing
- File upload
- Payment
- Full login system
- Backend server

It only syncs learning events generated by the existing local-first MVP.
