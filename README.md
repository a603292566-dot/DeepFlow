# DeepFlow MVP

DeepFlow 是一个基于 AI 的学习者身份、学习启动与成长证据系统。

## 本地运行

### PWA 网页版本

DeepFlow 当前支持作为可安装网页应用分发。部署到公网 HTTPS 后，用户可以在 iPhone、Android、Windows 和 Mac 上通过浏览器打开，并添加到主屏幕或安装为应用。

本地预览：

```bash
python3 -m http.server 4173
```

然后打开 `http://localhost:4173`。

部署构建：

```bash
npm run build
```

输出目录：

```text
dist
```

Vercel 配置：

```text
Build Command: npm run build
Output Directory: dist
Install Command: npm install
```

Netlify 配置：

```text
Build command: npm run build
Publish directory: dist
```

### macOS 桌面 App

当前项目可以构建为本地 macOS 程序，不需要打开 Chrome 或 Safari。

```bash
bash scripts/build-macos-app.sh
open build/macos/DeepFlow.app
```

也可以双击项目根目录中的 `open-deepflow.command`。它会重新构建并打开 DeepFlow，适合开发期间使用。

### 简单浏览器预览

如果只想用本地服务预览：

```bash
python3 -m http.server 4173
```

然后打开 `http://localhost:4173`。

DeepFlow 不直接充当完整课程平台，也不是传统内容平台、打卡工具或任务管理器。DeepFlow 的核心价值是：帮助用户创建学习者角色，选择学习方向与学习目标，生成适配外部 LLM 的学习教练 Prompt，并通过本地计时、启动阶梯、成长证据与 EXP 反馈，降低学习启动阻力，强化学习者身份。

## 一句话定义

DeepFlow 帮助用户用最低启动成本开始学习，通过 AI Prompt 调度外部 LLM 完成学习过程，并把每一次学习行为转化为可见的成长证据、EXP 和角色卡反馈。

## MVP 核心用户流程

1. 用户本地打开 DeepFlow。
2. 系统展示首页：`创建你的学习者角色`。
3. 用户点击开始。
4. 系统展示学习者身份调查问卷。
5. 用户填写并提交问卷。
6. 系统根据问卷结果生成学习者身份、初始等级与角色卡。
7. 系统展示：`开始学习`。
8. 用户点击开始学习。
9. 系统展示学习内容选择：`语言`、`数学`、`编程`。
10. 用户选择学习内容。
11. 系统展示学习目标选择，例如：德语 C1、微积分、Python 入门。
12. 用户选择学习目标。
13. 系统根据用户身份、学习内容、目标等级、当前模式与历史学习档案生成外部 LLM Prompt。
14. 系统展示：`复制 Prompt` 与 `开始学习`。
15. 用户点击开始学习，本地计时开始。
16. 用户前往外部 LLM，例如 ChatGPT、Claude、Gemini、Kimi、通义千问等，粘贴 Prompt 并完成学习。
17. DeepFlow 本地保持计时，并展示：`完成轻启动`。
18. 用户完成学习后回到 DeepFlow，点击完成轻启动。
19. 系统记录时间、模块、Prompt、目标、学习模式和成长证据。
20. 系统询问：`是否继续深度学习？`
21. 如果用户选择继续，系统生成下一阶段 Prompt，并继续计时。
22. 如果用户选择结束，系统结算本次学习，生成 EXP，更新角色卡，展示本次成长反馈。

## MVP 必须包含的功能

### 1. 学习者身份问卷

问卷是用户进入 DeepFlow 的第一步。问卷必须内置在产品中，不能只依赖外部表单。

问卷用于生成：

- 主身份
- 副身份
- 八项学习参数
- 初始角色等级
- 初始经验加成方向

### 2. 学习者身份生成

前台身份包括：

- 旅行者：学习者身份正在形成
- 探险家：探索驱动型学习者
- 学者：原理思维 / 研究型学习者
- 构筑者：系统建构型学习者
- 实干家：行动导向型学习者
- 大师：综合型高阶学习者

身份相对稳定，成长主要通过等级、EXP 和成长证据体现，不要把用户频繁改成不同身份。

### 3. 极简角色卡

角色卡负责正反馈与成长可视化，不负责长篇分析。

默认展示：

- 身份
- 副身份
- 等级
- 当前 EXP / 升级所需 EXP
- 经验条
- 距离下一级所需 EXP
- 近期 EXP 来源及具体数值
- 当前经验加成

详细身份解释、参数画像与成长建议放入可选入口：`查看身份报告`。

### 4. 学习内容选择

发布版 MVP 只支持三个学习内容大类：

- 语言
- 数学
- 编程

不要在第一版开放过多分类，避免认知负荷过高。

### 5. 学习目标选择

每个大类提供预设目标。

语言示例：

- 德语 C1
- 德语 B2
- 英语 C1
- 英语 B2
- 法语入门

数学示例：

- 微积分
- 线性代数
- 概率论
- 统计学基础

编程示例：

- Python 入门
- 前端开发
- AI 辅助编程
- 产品代码理解

### 6. AI Prompt 生成

Prompt 生成是 DeepFlow MVP 的核心功能。

DeepFlow 不直接承担所有教学内容，而是生成高质量学习教练 Prompt，让用户复制到外部 LLM 中执行。

Prompt 必须根据以下信息生成：

- 用户主身份
- 用户副身份
- 学习内容
- 学习目标
- 当前模式：轻启动 / 标准深化 / 深度学习 / 重启
- 当前状态
- 历史成长证据
- 当前经验加成方向
- 模块偏好
- 复习队列
- 未掌握内容

### 7. 本地计时

用户点击 `开始学习` 后，系统本地开始计时。

必须记录：

- 开始时间
- 结束时间
- 学习时长
- 学习内容
- 学习目标
- 学习模式
- 是否完成
- 是否继续深化

### 8. 启动阶梯

DeepFlow 默认使用启动阶梯：

- 轻启动：5–10 分钟
- 标准深化：20–30 分钟
- 深度学习：40 分钟以上

第一次开始时，系统只给轻启动 Prompt。完成轻启动后，用户自主选择是否继续深化。

### 9. 成长证据与 EXP

EXP 来源于成长证据，而不是单纯打卡。

成长证据包括：

- 完成问卷
- 创建身份
- 完成轻启动
- 完成标准学习
- 完成深度学习
- 完成反馈
- 完成结构化输出
- 完成知识迁移
- 完成项目推进
- 中断后重启

### 10. 可选身份报告

身份报告不是默认首页内容。用户可主动点击查看。

身份报告包括：

- 主身份解释
- 副身份解释
- 参数画像
- 核心优势
- 成长空间
- 经验加成逻辑
- 学习建议

## DeepFlow 不做什么

MVP 不做：

- 社交系统
- 内容信息流
- 完整课程库
- 视频课程
- 排行榜
- 强制打卡
- 大型 Dashboard
- 复杂动画
- 复杂任务管理器

DeepFlow 第一版只验证核心闭环：

问卷 → 身份 → 角色卡 → 学习内容选择 → 目标选择 → 生成 Prompt → 外部 LLM 学习 → 本地计时 → 完成轻启动 → 是否继续深化 → EXP 结算 → 角色卡更新

## 产品原则

### 低启动负荷

用户不需要手动拆解任务。系统负责把复杂目标压缩成一个低负荷入口。

### 用户选择权

系统不命令用户，而是展示可选路径。完成轻启动后，用户可以选择结束并结算，也可以选择继续深化。

### 成长证据优先

DeepFlow 不奖励打卡本身，而奖励具体成长证据。

### 信息分层

首页只展示角色卡和核心操作。详细信息通过可选按钮进入。

### 成长语言优先

系统不使用缺陷标签。低分参数必须转化为发展性描述。
