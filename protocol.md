# DeepFlow Protocol V3

本文件定义 DeepFlow 发布版 MVP 的用户流程、问卷协议、启动阶梯协议、Prompt 协议、结算协议与角色卡协议。

## 1. 首次进入流程

1. 用户本地打开 DeepFlow。
2. 系统展示欢迎页。
3. 页面核心按钮：`创建你的学习者角色`。
4. 用户点击后进入问卷。

## 2. 学习者身份问卷流程

系统展示学习者身份调查问卷。

问卷说明：

> 本问卷用于生成你的 DeepFlow 学习者角色。
> 没有标准答案。
> 请按照过去半年真实情况作答。
> 不要回答理想中的自己。
> 请回答现实中的自己。

选项：

- 1 = 非常不同意
- 2 = 不同意
- 3 = 一般
- 4 = 同意
- 5 = 非常同意

用户提交后，系统计算八项参数并生成身份。

## 3. 问卷内容

### 探索驱动力 explorationDrive

Q1. 我经常主动想了解陌生领域。  
Q2. 遇到新概念时，我会想知道它背后的原理。  
Q3. 我喜欢把学习看成一种探索。  
Q4. 我愿意尝试不熟悉但有价值的主题。  
Q5. 我经常因为好奇而开始学习某个东西。  

### 执行稳定性 executionStability

Q6. 我能比较稳定地按计划完成学习。  
Q7. 即使状态一般，我也能完成一个小学习任务。  
Q8. 我很少因为启动困难而推迟学习。  
Q9. 我能把学习任务持续推进到结束。  
Q10. 我能在中断后较快恢复学习。  

### 系统建构能力 systemBuilding

Q11. 我习惯把零散知识整理成结构。  
Q12. 学习新内容时，我会主动建立框架。  
Q13. 我会思考不同知识点之间的层级关系。  
Q14. 我喜欢制作笔记、图谱或知识结构。  
Q15. 我能把复杂内容拆成清晰模块。  

### 原理思维 principleThinking

Q16. 我学习时更关心“为什么”而不只是“是什么”。  
Q17. 我习惯追问一个现象背后的机制。  
Q18. 我喜欢从底层原理理解问题。  
Q19. 我会尝试用模型解释现实问题。  
Q20. 我不满足于只记住结论。  

### 成长信念 growthBelief

Q21. 我相信学习能力可以通过训练提升。  
Q22. 即使当前水平不高，我也相信可以逐步成长。  
Q23. 我认为错误反馈是成长的一部分。  
Q24. 我愿意通过小步骤积累能力。  
Q25. 我相信持续行动会改变学习状态。  

### 情绪稳定性 emotionalStability

Q26. 学习遇到困难时，我能保持基本稳定。  
Q27. 我不会因为一次失败就否定自己的学习能力。  
Q28. 我能接受学习过程中的波动。  
Q29. 当学习不顺利时，我能重新调整。  
Q30. 我能区分暂时状态和长期能力。  

### 学习者认同 learnerIdentity

Q31. 我认为自己是一个正在成长的学习者。  
Q32. 学习是我身份的一部分。  
Q33. 我愿意长期建设自己的学习能力。  
Q34. 我希望自己成为更稳定的学习者。  
Q35. 当我完成学习时，我会感到自己更接近理想状态。  

### 知识连接能力 knowledgeConnection

Q36. 我经常把不同领域的知识联系起来。  
Q37. 我能从一个领域迁移经验到另一个领域。  
Q38. 我喜欢发现概念之间的隐藏联系。  
Q39. 我会用一个领域的模型解释另一个领域的问题。  
Q40. 我学习时经常产生跨学科联想。  

## 4. 问卷评分逻辑

每题 1–5 分。

每个参数包含 5 题。

原始分范围：5–25。

转换为 0–100：

```ts
score100 = Math.round(((rawScore - 5) / 20) * 100)
```

## 5. 身份判定协议

### 学者 scholar

倾向条件：

- principleThinking >= 75
- knowledgeConnection >= 75
- explorationDrive >= 65

### 探险家 explorer

倾向条件：

- explorationDrive >= 80
- knowledgeConnection >= 65

### 构筑者 builder

倾向条件：

- systemBuilding >= 75
- principleThinking >= 60

### 实干家 doer

倾向条件：

- executionStability >= 75
- learnerIdentity >= 60

### 旅行者 traveler

倾向条件：

- learnerIdentity < 60
- 或 executionStability < 50
- 且其他维度没有形成明显主导身份

### 大师 master

倾向条件：

- 至少 6 个参数 >= 75
- executionStability >= 70
- learnerIdentity >= 70

多身份同时满足时，主身份取最强身份，副身份取第二强身份。

## 9. Prompt 生成流程

系统根据以下信息生成 Prompt：

- 用户身份
- 副身份
- 学习内容
- 学习目标
- 当前模式
- 当前状态
- 历史学习记录
- 近期经验加成
- 复习队列
- 模块偏好

Prompt 不应要求用户完成过大任务。

## 10. 启动阶梯协议

### 轻启动 light_start

预计时长：5–10 分钟。

目标：降低启动阻力，生成第一条成长证据。

规则：

- 不给完整计划
- 不要求长输出
- 不给过多材料
- 只给一个小任务

### 标准深化 standard

预计时长：20–30 分钟。

触发条件：用户完成轻启动并选择继续。

目标：更完整的理解、练习和输出。

### 深度学习 deep

预计时长：40 分钟以上。

触发条件：用户完成标准深化并选择继续。

目标：知识迁移、体系化整理、模型构建或完整输出。

### 重启 restart

触发条件：用户中断学习后重新开始。

目标：恢复连续性，而不是补偿损失。

## 11. 本地计时协议

用户点击 `开始学习`：

- 记录 startTime
- 启动计时器
- 创建 LearningSession

用户点击 `完成轻启动`：

- 记录 endTime
- 计算 durationMinutes
- 保存 Prompt
- 生成 GrowthEvidence
- 计算 EXP
- 展示是否继续深化

## 12. 继续深化协议

完成轻启动后展示：

- 轻启动完成
- 获得 EXP
- 成长证据
- 是否继续学习？

选项：

- 继续学习
- 结束并结算

如果继续：生成下一阶段 Prompt。

如果结束：结算本次学习并更新角色卡。

## 13. 结算协议

结算展示：

- 学习内容
- 学习目标
- 学习模式
- 实际时长
- 获得成长证据
- 获得 EXP
- 当前角色卡

反馈必须简短、具体、数字化。

## 14. 角色卡协议

默认角色卡只展示：

- 身份
- 副身份
- 等级
- EXP
- 经验条
- 距离升级
- 近期 EXP 来源
- 当前经验加成

不展示长篇分析。

## 15. 可选身份报告

用户可点击 `查看身份报告`。

报告展示：

- 身份解释
- 参数画像
- 核心优势
- 成长空间
- 经验加成逻辑

报告必须使用成长语言，不使用缺陷标签。
